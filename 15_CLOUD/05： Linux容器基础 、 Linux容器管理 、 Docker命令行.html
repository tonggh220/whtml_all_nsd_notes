1.1 问题

本案例要求为主机安装docker软件，具体要求如下：

步骤二：使用镜像，运行容器

1）运行容器

可以通过docker help run或者man docker-run查看帮助。

法法格式如下：docker run -参数 镜像名称：镜像标签 启动命令

run命令 = 创建 + 启动 + 进入

docker run 命令的重要参数如下

  • 参数 -i，交互式
  • 参数 -t，终端
  • 参数 -d，后台运行
  • 参数 --name 容器名字

启动 centos 容器，并进入容器

copytextpop-up
  1. [root@node-0001 ~]# docker run -it --name myos centos:latest /bin/bash
[root@node-0001 ~]# docker run -it --name myos centos:latest /bin/bash

使用docker命令启动容器，可以通过提示符判定自己是否进入容器

  2. [root@10d70724abf2 /]# ps –ef                   #可以看到命令提示符已经变了
  3. UID        PID  PPID  C STIME TTY          TIME CMD
  4. root         1     0  0 12:52 ?        00:00:00 /bin/bash
  5. root        14     1  0 12:53 ?        00:00:00 ps -ef
[root@node-0001 ~]#  docker run -it centos:latest /bin/bash
[root@10d70724abf2 /]# ps –ef                   #可以看到命令提示符已经变了

copytextpop-up
  1. [root@node-0001 ~]# docker ps  [ -a ] [ -q ]
  2. #[]代表可选参数，可以使用-a或-q也可以不适用

[root@node-0001 ~]# docker rm $(docker ps -aq)   #删除已经启动所有容器

6 案例6：练习容器的执行方式

6.1 问题

本案例要求理解容器的执行方式：

  1. 前台服务
  2. 后台服务
  3. 创建一个 centos 的容器，并为他设置 yum 源

6.2 步骤

实现此案例需要按照如下步骤进行。

步骤一：概念

容器启动服务的方式

前台服务（-it）：一般是能与用户交互的程序，比如 /bin/bash、/bin/sh 等

后台服务（-itd）：一般是一个程序服务，比如 apache、nginx、redis 等

步骤二：命令练习

copytextpop-up
  1. [root@node-0001 ~]# docker run -d  centos:latest        # 失败

  3. [root@node-0001 ~]# docker ps -a
  4. CONTAINER ID   IMAGE              COMMAND   CREATED             STATUS      NAMES
  5. c2219228afc1    centos:latest   "/bin/bash"   5 seconds ago    Exited (0)      xx
  6. [root@node-0001 ~]# docker run -itd  centos:latest        # 成功
[root@node-0001 ~]# docker run -d  centos:latest		# 失败

c2219228afc1    centos:latest   "/bin/bash"   5 seconds ago    Exited (0)      xx
[root@node-0001 ~]# docker run -itd  centos:latest		# 成功

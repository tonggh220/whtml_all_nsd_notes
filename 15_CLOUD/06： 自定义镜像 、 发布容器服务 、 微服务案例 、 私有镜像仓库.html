2）通过Dockerfile创建自定义镜像。

通过docker build命令可以根据 Dockerfile 里的内容生成镜像

编写Dockerfile的语法格式如下：

FROM: 基础镜像

RUN: 制作镜像时执行的命令，可以有多个

ADD: 复制文件到镜像，自动解压

COPY: 复制文件到镜像，不解压

EXPOSE: 声明开放的端口

ENV: 设置容器启动后的环境变量

WORKDIR: 定义容器默认工作目录（等于cd）

CMD: 容器启动时执行的命令，仅可以有一条CMD

具体操作流程如下：

创建目录 mkdir mybuild

在目录中编写 Dockerfile

生成镜像

docker build -t 镜像名称:标签 Dockerfile所在目录

3 案例3：发布容器服务

3.1 问题

本案例练习测试docker的端口绑定和主机卷映射服务，分别实现以下目标：

  1. 通过映射端口对外发布服务
  2. 创建 /var/webroot、 /var/webconf
  3. 在/var/webroot下创建自定义首页
  4. 拷贝apache的配置文件httpd.conf到 /var/webconf
  5. 修改监听端口为 8080
  6. 启动apache，使用/var/webconf/配置文件替换容器内的配置文件，使用/var/webroot/替换容器首页，并绑定宿主机80端口，访问验证

3.2 步骤

实现此案例需要按照如下步骤进行。

步骤一：通过映射端口发布服务（可以在node-0001或者node-0002操作）

我们使用-p参数把容器端口和宿主机端口绑定，

一个宿主机端口只能绑定一个容器服务。

例如:把宿主机变成 apache

copytextpop-up
  1. [root@node-0001 ~]# docker run -itd -p 80:80 myos:httpd
  2. # 将真机的80端口映射到容器的80端口
  3. # 这样浏览器访问真机（宿主机）的80端口就等于是访问容器的80端口
[root@node-0001 ~]# docker run -itd -p 80:80 myos:httpd
# 将真机的80端口映射到容器的80端口
# 这样浏览器访问真机（宿主机）的80端口就等于是访问容器的80端口

例如:把宿主机变成nginx（注意把上面的容器关闭后再做）

copytextpop-up
  1. [root@node-0001 ~]# docker run -itd -p 80:80  docker.io/nginx:latest
[root@node-0001 ~]# docker run -itd -p 80:80  docker.io/nginx:latest

步骤二：容器共享卷

Docker容器不适合保存任何数据，数据文件与配置文件频繁更改，修改多个容器中的数据非常困难，多容器之间有数据共享、同步需求，重要数据在容器内不方便管理易丢失，解决这些问题请使用主机卷映射功能。

Docker可以映射宿主机文件或目录到容器中：

  • 目标对象不存在就自动创建
  • 目标对象存在就直接覆盖掉
  • 多个容器可以映射同一个目标对象来达到数据共享的目的
  • 启动容器时，使用 -v 映射参数（可有多个）

语法格式如下：

docker run -itd -v 宿主机对象:容器内对象 myos:latest

apache使用宿主机中的配置文件和网页家目录

copytextpop-up
  1. [root@node-0001 ~]#  docker run -itd myos:httpd
  2. [root@node-0001 ~]#  mkdir /var/{webroot,webconf}
  3. [root@node-0001 ~]# docker cp \
  4. 2422cc74d7b6:/etc/httpd/conf/httpd.conf /var/webconf/
  5. [root@localhost ~]# docker rm - f  2422cc74d7b6
  6. # 注意这里的 2422cc74d7b6 为容器ID ，不要照抄！
  7. # 这里是启动一个httpd的容器，把容器中的配置文件拷贝到真机的/var/webconf/目录
  8.  
  9. [root@node-0001 ~]# echo "i am apache" >>/var/webroot/index.html
  10. [root@node-0001 ~]# vim /var/web/conf/httpd.conf
  11. #在真机修改httpd.conf的配置文件，修改端口为8080
  12.  
  13. [root@node-0001 ~]# docker run -itd \
  14. -v /var/webroot:/var/www/html myos:httpd
  15. # 将真机（宿主机）的/var/webroot目录映射到容器中的/var/www/html目录
  16.  
  17. [root@node-0001 ~]# docker run -itd \
  18. -v /var/webconf/httpd.conf:/etc/httpd/conf/httpd.conf myos:httpd
  19. # 将真机（宿主机）的/var/webconf/httpd.conf文件映射到容器中的
[root@node-0001 ~]#  docker run -itd myos:httpd
[root@node-0001 ~]#  mkdir /var/{webroot,webconf}
[root@node-0001 ~]# docker cp \
2422cc74d7b6:/etc/httpd/conf/httpd.conf /var/webconf/
[root@localhost ~]# docker rm - f  2422cc74d7b6
# 注意这里的 2422cc74d7b6 为容器ID ，不要照抄！
# 这里是启动一个httpd的容器，把容器中的配置文件拷贝到真机的/var/webconf/目录

[root@node-0001 ~]# echo "i am apache" >>/var/webroot/index.html
[root@node-0001 ~]# vim /var/web/conf/httpd.conf
#在真机修改httpd.conf的配置文件，修改端口为8080

[root@node-0001 ~]# docker run -itd \
-v /var/webroot:/var/www/html myos:httpd
# 将真机（宿主机）的/var/webroot目录映射到容器中的/var/www/html目录

[root@node-0001 ~]# docker run -itd \
-v /var/webconf/httpd.conf:/etc/httpd/conf/httpd.conf myos:httpd
# 将真机（宿主机）的/var/webconf/httpd.conf文件映射到容器中的

结合上面的端口映射的案例，可以将真机的80和容器的8080进行映射，映射后就可以通过访问真机的80端口访问容器的8080了。

步骤一：php-fpm镜像创建（在node-0001操作）

因为我们想将Nginx和PHP做成两个独立的镜像，默认PHP的配置文件默认禁止远程访问，这样nginx就无法连接PHP，我们在制作php-fpm镜像时需要修改PHP的配置文件。

先在真机（宿主机）安装php-fpm，在真机直接修改配置文件，方便后面做镜像。

copytextpop-up
  1. [root@node-0001 ~]# yum install  php-fpm
  2. [root@node-0001 ~]# mkdir php  ;   cd  php
  3. [root@node-0001 php]# cp /etc/php-fpm.d/www.conf    ./
  4. [root@node-0001 php]# vim www.conf
  5. listen = 0.0.0.0:9000                          #修改该行的内容
  6. ... ... 
  7. ;listen.allowed_clients = 127.0.0.1          #删除这一行整行内容
[root@node-0001 ~]# yum install  php-fpm
[root@node-0001 ~]# mkdir php  ;   cd  php
[root@node-0001 php]# cp /etc/php-fpm.d/www.conf    ./
[root@node-0001 php]# vim www.conf
listen = 0.0.0.0:9000                          #修改该行的内容
... ... 
;listen.allowed_clients = 127.0.0.1          #删除这一行整行内容

我们还希望在php-fpm镜像里面放一个动态网页，动态网页内容以及提前共享到云盘 ，在第四阶段kubernetes/docker-images/info.php，请提前将该文件拷贝到实验虚拟机的/root/php目录中。

编写Dockerfile文件，制作自定义镜像

copytextpop-up
  1. [root@node-0001 php]#  vim Dockerfile
  2. FROM myos:latest
  3. RUN  yum install -y php-fpm
  4. COPY www.conf /etc/php-fpm.d/www.conf
  5. EXPOSE 9000
  6. WORKDIR /usr/local/nginx/html
  7. COPY info.php info.php
  8. CMD ["/usr/sbin/php-fpm", "--nodaemonize"]
  9. # 备注：
  10. # RUN可以在自定义镜像中安装php-fpm软件
  11. # EXPOSE指定容器开放的端口为9000
  12. # COPY指令可以将真机的配置文件www.conf拷贝到自定义镜像中
  13. # WORKDIR可以设置自定义镜像的工作目录为/usr/local/nginx/html
  14. # COPY指令将真机的info.php这个动态网页拷贝到自定义镜像中
  15. # CMD指令指定启动容器时要运行的命令
[root@node-0001 php]#  vim Dockerfile
FROM myos:latest
RUN  yum install -y php-fpm
COPY www.conf /etc/php-fpm.d/www.conf
EXPOSE 9000
WORKDIR /usr/local/nginx/html
COPY info.php info.php
CMD ["/usr/sbin/php-fpm", "--nodaemonize"]
# 备注：
# RUN可以在自定义镜像中安装php-fpm软件
# EXPOSE指定容器开放的端口为9000
# COPY指令可以将真机的配置文件www.conf拷贝到自定义镜像中
# WORKDIR可以设置自定义镜像的工作目录为/usr/local/nginx/html
# COPY指令将真机的info.php这个动态网页拷贝到自定义镜像中
# CMD指令指定启动容器时要运行的命令
